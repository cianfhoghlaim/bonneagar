# Dagster (Oideachais Unified)

## Overview
Dagster is an open-source data orchestration platform for building, testing, and running data pipelines. This stack runs a custom-built `ghcr.io/cianfhoghlaim/dagster:latest` image containing the entire Oideachais data platform — DLT ingestion pipelines, curriculum extraction assets, and Celtic education data transformations. The stack includes both the webserver (`dagster-unified`, port 3335→3000) and the daemon (`dagster-daemon`) for schedules and sensors.

## Why This Matters for Kings' College Galway
Dagster is the beating heart of the Celtic education data platform. It orchestrates every step of the curriculum ingestion lifecycle: DLT pipelines that scrape Irish and UK curriculum websites, Firecrawl-based extraction jobs that parse Leaving Cert exam papers, DuckLake materialization of Parquet data into the Garage S3 lakehouse, and LanceDB vector embeddings for semantic search. The daemon runs scheduled jobs that keep curriculum data fresh, while the webserver provides the UI for monitoring pipeline health, debugging failed runs, and triggering ad-hoc materializations. Every study asset and flashcard generated by the platform traces its lineage back to a Dagster asset.

## Key Features
- Custom Docker image with the full Oideachais data platform baked in
- Asset-based pipeline orchestration with DuckLake/Iceberg catalog integration
- Garage S3 object storage for Parquet data with LanceDB vector embeddings
- Langfuse observability for LLM call tracing within pipeline steps
- Daemon process for schedules, sensors, and automated run queues
- 4 CPU / 4GB memory allocation for the webserver, 2 CPU / 2GB for the daemon

## Deployment

### Docker Compose (Local Development)
```bash
cd infrastructure/stacks/engineering/dagster
docker compose up -d
```

### Docker Compose (Production with Locket Secret Injection)
```bash
cd infrastructure/stacks/engineering/dagster
docker compose -f compose.yaml -f sidecar.yaml -f pangolin.yaml up -d
```

### Komodo (GitOps)
This stack is deployed via Komodo on arm1-oci. Komodo syncs from the Forgejo repository and applies `compose.yaml` + `sidecar.yaml`. No `.env` file is needed — Locket resolves all secrets from the Infisical `dev-baile` vault at runtime.

## Environment Variables
| Variable | Required | Description | Default |
|:--|:--|:--|:--|
| `DAGSTER_PORT` | No | Host port for the webserver | `3335` |
| `DAGSTER_HOME` | No | Dagster home directory | `/opt/dagster/home` |
| `USE_DUCKLAKE` | No | Enable DuckLake integration | `true` |
| `DUCKLAKE_POSTGRES_HOST` | No | Lakehouse Postgres host | `lakehouse-postgres` |
| `DUCKLAKE_POSTGRES_PORT` | No | Lakehouse Postgres port | `5432` |
| `DUCKLAKE_POSTGRES_USER` | No | Lakehouse Postgres user | `lakekeeper` |
| `DUCKLAKE_POSTGRES_PASSWORD` | Yes | Lakehouse Postgres password | — |
| `DLT_ENVIRONMENT` | No | DLT runtime environment | `production` |
| `AWS_ACCESS_KEY_ID` | Yes | Garage S3 access key | — |
| `AWS_SECRET_ACCESS_KEY` | Yes | Garage S3 secret key | — |
| `AWS_ENDPOINT_URL` | No | Garage S3 endpoint URL | `http://lakehouse-garage:3900` |
| `AWS_REGION` | No | S3 region | `garage` |
| `ANTHROPIC_API_KEY` | No | Anthropic API key for LLM steps | — |
| `OPENAI_API_KEY` | No | OpenAI API key for LLM steps | — |
| `LANGFUSE_PUBLIC_KEY` | No | Langfuse public key | — |
| `LANGFUSE_SECRET_KEY` | No | Langfuse secret key | — |
| `LANGFUSE_HOST` | No | Langfuse host URL | `https://langfuse.cianfhoghlaim.ie` |

## Access
- **URL**: `https://dagster.cianfhoghlaim.ie` (private, Pangolin Member role required)
